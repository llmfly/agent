/api/v1/conversations 与 /api/threads 接口映射关系

基于当前代码静态分析生成。v1 总路由前缀为 /api/v1，其中 conversations 路由前缀为 /conversations；内部大多数能力以 conversation_id = thread_id 的方式复用 Gateway thread/run/upload/artifact 能力。

目录

总体结论 概念映射 会话 CRUD 消息与运行 状态与历史 上传文件 产物与建议 v1 其它资源覆盖 缺口与建议 代码来源

总体结论

当前 /api/v1/conversations 已覆盖前端聊天工作台的大部分 thread 能力：会话列表、创建、详情、更新、删除、消息、流式发送、运行列表、上传、状态、历史、token usage、建议问题、按路径读取 artifact。

需要特别关注两类不完全等价：一是 DELETE /api/v1/conversations/{id} 没有完全复用 DELETE /api/threads/{thread_id} 的本地文件清理语义；二是 raw run lifecycle、run events、完整 feedback 管理没有全部放在 conversations 子路径下。

概念映射

v1 概念threads 概念说明
conversation_idthread_idv1 对外称 conversation，内部直接使用同一 ID 访问 thread/run/checkpoint/upload/artifact。
agent_idassistant_idv1 DTO 中叫 agent，内部 run/thread 记录多用 assistant。
titledisplay_name / values.title / metadata.title创建/更新时写 thread store display_name；状态更新时 title 可能同步到 display_name。

runRunManager recordv1 将底层运行记录转成 RunDTO，字段更面向外部调用方。
artifactthread virtual path 或 v1 artifact DB recordconversations 下同时支持 conversation artifact 列表与 thread path 文件读取。

会话 CRUD

GET/api/v1/conversations间接映射

对应：POST /api/threads/search

用途：列出当前外部应用和外部用户可见的 conversations。

入参

Querylimit: 1-100，默认 20；offset: 默认 0；status: 可选。
隐式上下文ExternalContext 提供 app_id、external_user_id，用于 metadata 过滤。

出参

{
  "items": [ConversationDTO],
  "pagination": {"limit": 20, "offset": 0, "has_more": false}
}

差异：底层 /api/threads/search 返回 list[ThreadResponse]；v1 包装分页，并补充 last_message。

POST/api/v1/conversations间接映射

对应：POST /api/threads

用途：创建外部 conversation，同时创建 thread meta 和空 checkpoint。

入参

{
  "agent_id": "lead-agent",
  "title": "新会话",
  "metadata": {"key": "value"}
}

出参

{
  "conversation_id": "uuid",
  "agent_id": "lead-agent",
  "title": "新会话",
  "status": "idle",
  "last_message": null,
  "created_at": "...",
  "updated_at": "...",
  "metadata": {}
}

差异：底层允许传 thread_id 和 assistant_id；v1 自己生成 UUID，并将 external context 写入 metadata。

POST/api/v1/conversations/search间接映射

对应：POST /api/threads/search

入参

{
  "metadata": {"project": "demo"},
  "limit": 20,
  "offset": 0,
  "status": "idle"
}

出参

{"items": [ConversationDTO], "pagination": PaginationDTO}

差异：v1 自动叠加 app_id 和可选 external_user_id metadata 过滤。

GET/api/v1/conversations/{conversation_id}间接映射

对应：GET /api/threads/{thread_id}

入参

Path: conversation_id，即内部 thread_id。

出参

ConversationDTO

差异：底层会返回 values、interrupts；v1 只返回 conversation 元数据，并尝试 enrich artifact metadata。

PATCH/api/v1/conversations/{conversation_id}间接映射

对应：PATCH /api/threads/{thread_id}

入参

{"title": "重命名", "metadata": {"key": "value"}}

出参

ConversationDTO

差异：底层 patch 只 merge metadata；v1 还支持 title，并写入 thread store display_name。

DELETE/api/v1/conversations/{conversation_id}部分映射

对应：DELETE /api/threads/{thread_id}

入参

Path: conversation_id。

出参

HTTP 204 No Content。

差异：v1 删除 checkpoint 和 thread meta；底层 threads 删除还会清理 DeerFlow thread 本地目录。若外部 API 需要强一致清理，应补齐此语义。

出参

v1 其它资源对 threads 子能力的覆盖

部分 thread run 和 artifact 能力没有放在 /api/v1/conversations/{id} 下，而是放在 v1 独立资源下。

v1 API对应 threads API说明
GET /api/v1/runs/{run_id}GET /api/threads/{thread_id}/runs/{run_id}通过 run_id 反查 conversation_id。
POST /api/v1/runs/{run_id}/cancelPOST /api/threads/{thread_id}/runs/{run_id}/cancelv1 使用 body，底层使用 query。
GET /api/v1/runs/{run_id}/joinGET /api/threads/{thread_id}/runs/{run_id}/joinjoin existing run SSE。
GET /api/v1/runs/{run_id}/streamGET /api/threads/{thread_id}/runs/{run_id}/stream订阅已有 run stream。
POST /api/v1/runs/{run_id}/streamPOST /api/threads/{thread_id}/runs/{run_id}/stream支持 cancel-then-stream。
PUT /api/v1/runs/{run_id}/feedbackPUT /api/threads/{thread_id}/runs/{run_id}/feedbackupsert 当前用户反馈。
DELETE /api/v1/runs/{run_id}/feedbackDELETE /api/threads/{thread_id}/runs/{run_id}/feedback删除当前用户反馈。

缺口与建议

底层接口当前 v1 状态建议
POST /api/threads/{thread_id}/runsconversations 下无 raw create如果前端需要只创建后台 run、不等待、不流式，补 POST /api/v1/conversations/{id}/runs。
POST /api/threads/{thread_id}/runs/wait被 /messages 覆盖聊天场景如果要透传完整 RunCreateRequest，补 POST /api/v1/conversations/{id}/runs/wait。
POST /api/threads/{thread_id}/runs/stream被 /stream 覆盖聊天场景如果要兼容 LangGraph SDK 形态，补 POST /api/v1/conversations/{id}/runs/stream。
GET /api/threads/{thread_id}/runs/{run_id}v1 runs 独立资源已覆盖可选补 GET /api/v1/conversations/{id}/runs/{run_id}，提高路径对称性。
GET /api/threads/{thread_id}/runs/{run_id}/eventsv1 未封装建议补 GET /api/v1/conversations/{id}/runs/{run_id}/events，用于调试、审计、前端事件回放。
POST /api/threads/{thread_id}/runs/{run_id}/feedbackv1 未覆盖 message_id create如果前端要对具体消息反馈，建议补。
GET /api/threads/{thread_id}/runs/{run_id}/feedbackv1 未覆盖管理台或调试需要查看反馈列表时补。
GET /api/threads/{thread_id}/runs/{run_id}/feedback/statsv1 未覆盖需要展示满意度统计时补。
DELETE /api/threads/{thread_id}/runs/{run_id}/feedback/{feedback_id}v1 未覆盖需要管理员删除指定反馈记录时补。
DELETE /api/threads/{thread_id}v1 删除语义不完全等价建议确认是否要在 DELETE /api/v1/conversations/{id} 中补齐本地 filesystem cleanup。
