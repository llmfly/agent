# 公司洞察报告

## 能力边界

本 skill 从 `trade-report-generator` 复制并扩展而来，覆盖两件事：

1. 根据已计算好的企业海关交易指标，生成正式研报正文所需的分析段落。
2. 使用随 skill 打包的 Word 模板配置和渲染脚本，把结构化指标、图表路径和分析段落合成为 `.docx`。

不要在分析段落中发明数据。指标计算、图表生成和数据清洗应在外部完成；本 skill 接收结构化指标 JSON、图表文件和 agent 生成的章节文字，然后完成研报写作与文档装配。

## 资源

- `assets/word_template.json`：Word 报告模板配置，定义标题、作者、字体意图、章节、表格和图表位置。
- `assets/sample/sample_metrics.json`、`assets/sample/sample_analysis.md`、`assets/sample/charts/`：最小可跑样例，用于验证 skill 迁移后是否能生成 `.docx`。不要把这些样例当成真实业务结论。
- `assets/sample/sample_report.docx`：由上述样例数据生成的参考 Word 成品，用于迁移后对照排版和内容结构。
- `scripts/render_company_insight_docx.py`：把指标 JSON 和章节分析 Markdown 渲染成 `.docx`。
- `scripts/calculate_trade_metrics.py`：将原始海关 Excel 数据转换为结构化指标 JSON（实现 `references/data-processing-logic.md`）。
- `references/comprehensive-metrics-logic.md`：高级指标计算逻辑，包含国家增速、买家开放度及市场集中度算法，用于生成高质量证据支撑的研报。
- `references/automotive-analysis-logic.md`：汽车行业专项贸易解读逻辑，涵盖 NEV 转型信号、产业链输出及市场准入路径分析。
- `references/excel-schema-example.md`：记录了海关交易数据 Excel 的标准列名及探测策略，用于快速对齐数据结构。
- `references/excel-schema-variants.md`：汇总了不同数据源中常见的列名变体映射表，用于处理非标准 schema 文件。

## 推荐流程

**执行清单 (Execution Checklist) —— 严禁仅输出计划/步骤/Stub，必须端到端交付：**
1. **主体识别与数据探测 (Highest Priority)：** 面对 Excel 文件，必须先调用 `pd.ExcelFile(path).sheet_names` 确认交易数据所在 Sheet。**核心要求：绝不能直接用用户提供的名称过滤数据。** 必须先通过 `df['供应商'].unique()` 等方式探测数据集中的实际名称，利用 `str.contains` 进行关键词匹配以锁定完整的法律实体名称（例如将“江淮汽车” $\rightarrow$ “安徽江淮汽车集团股份有限公司”），确保不会因名称差异导致数据被误判定为缺失。建议打印前几行数据以确认实际列名，防止 `KeyError`。如果用户未明确指定企业且工作簿 `README` 包含 `目标企业`，可以使用该目标企业；如果用户明确指定的企业与 `README` 目标企业不一致，必须停止并说明二者不匹配，不能擅自切换目标企业。不要仅因市场样本、竞品或 HS 描述中出现石油、化工等词就判定汽车企业文件错误；应以显式目标企业行是否存在、金额是否非零作为是否继续生成的主要依据。
2. **计算指标：** 参照 `references/excel-to-metrics-impl.md` 的实现逻辑计算指标，并使用 `clean_numpy` 递归清理所有 NumPy 类型，确保所有 `json.dump` 调用均不报错。
3. **撰写正文：** 严格遵循“结论 → 证据 → 解释 → 含义”的研报逻辑，禁止使用空洞形容词。正文中不要输出 LaTeX 箭头语法片段。
4. **文档渲染：** 调用 `scripts/render_company_insight_docx.py` 生成 `.docx`。
5. **结果验证：** 在返回结果前，必须通过代码（如 `os.path.exists`）确认 `.docx` 文件已实际生成在目标路径。
6. **绝对路径交付：** 在最终响应中必须提供 `report_path` 字段，值为生成的 `.docx` 文件的绝对路径。绝对禁止要求用户开启新会话或仅提供执行计划。

**场景 A：已有指标数据**
1. **数据确认：** 检查用户是否提供了结构化指标 JSON。若未提供，但提供了原始海关数据（如 Excel），请参考 `references/data-processing-logic.md` 计算结构化指标。
...

**关键避坑：** 
1. **数据类型兼容性：** 在使用 pandas 计算指标并保存为 JSON 时，必须将 NumPy 类型（如 `int64`, `float64`）转换为 Python 原生类型（`int`, `float`），否则 `json.dump` 会抛出 `TypeError`。建议编写一个递归清理函数处理所有 NumPy 标量及数组。
2. **脚本语法检查：** 某些版本的 `calculate_trade_metrics.py` 脚本可能包含错误的转义引号（如 `\"Concentrated\"`），导致 `SyntaxError`。在执行前请确认脚本语法正确，或在发现错误时立即使用 `patch` 工具修复。
2. 计算结构化指标，生成一个 JSON 文件。字段应尽量包含本 skill 的“输入”章节列出的键。

**场景 B：无指标数据（纯文本请求）**
1. 使用 web_search 等工具调研企业的出口规模、核心产品、目标市场及竞争对手。
2. 将调研结果整理成结构化的指标清单（内部草稿或 JSON），确保每个判断都有数据支撑。
3. 参照“场景 A”后续步骤生成分析段落。
2. 生成 PNG 或 JPEG 图表，并在 JSON 中提供图表路径字段，例如 `chart_monthly_trend`、`chart_country_amount`、`chart_product_amount`。Word 输出不直接嵌入 PDF 图。
3. 按“输出格式”生成章节分析 Markdown。每个章节必须用 `###` 标题分隔。
4. 调用渲染脚本生成文档：

```bash
SKILL_DIR="${HERMES_HOME:-$HOME/.hermes}/skills/company-insight-report"

python "$SKILL_DIR/scripts/render_company_insight_docx.py" \
  --metrics-json outputs/company_metrics.json \
  --analysis-md outputs/company_analysis.md \
  --output-dir outputs \
  --output-name "report_name" \
  --no-timestamp
```

迁移或安装 skill 后，先跑随 skill 打包的样例验证环境：


## Docker 运行要求

Hermes 导入 skill 后，只要容器内有 Python 3，就可以运行 `scripts/render_company_insight_docx.py` 生成 Word 文档。脚本直接写 OpenXML `.docx` 压缩包，不需要 TeX Live，不需要 LibreOffice，也不需要联网安装依赖。

注意：Word 内嵌图表只支持 PNG/JPEG。若上游图表是 PDF，应先在上游转换为 PNG，再写入 metrics JSON。

## 文档生成约定

- 渲染脚本只负责文档装配，不调用大模型。
- 默认使用时间戳命名：`公司名_出海贸易分析报告_YYYYmmdd_HHMMSS.docx`。
- 分析段落中的 Markdown 加粗 `**短标题。**` 会转换为 Word 加粗文本。
- 表格数据来自指标 JSON，正文判断来自分析 Markdown。两者应保持一致。
- **最终交付物：任务完成后，必须在最终响应中提供一个包含 `report_path` 字段的 JSON 或结构化文本，其值为生成的 `.docx` 文件的绝对路径。**
- 生产数据不要放进 skill。真实交易明细、数据库连接、客户名单和大体量模拟明细应由外部项目、数据库或运行参数提供；skill 内只保留最小样例数据和模板资产。

## 角色

你是一位企业出海贸易研究员。你的任务是根据已计算好的指标和表格，为一份固定的企业出海贸易分析报告模板，生成可直接进入正式研报正文的章节分析。Word 文档结构、表格和图表由渲染脚本完成，你只需输出**分析性文字段落**。


## 数据预处理与指标提取要点

在执行指标计算前，必须处理海关数据中常见的非标准情况：

1. **企业名称模糊匹配**：用户提供的公司名可能是简称（如“江淮汽车”），而数据集中为全称（如“安徽江淮汽车集团股份有限公司”）。应先通过 `str.contains()` 进行关键词检索，确认数据集中实际存在的企业名称，再进行过滤。
2. **灵活列名映射**：海关数据的列名经常变动（例如“供应商” $\leftrightarrow$ “主体企业”，“总价” $\leftrightarrow$ “金额美元”）。在编写计算脚本时，应建立一个映射字典（Mapping Dictionary）将原始列名统一转换为内部标准字段（如 `company`, `amount`, `date`），以增强脚本的鲁棒性。
3. **数据清洗**：确保交易日期转换为 `datetime` 格式，并处理金额列中的非数值字符，防止计算崩溃。

## 研报写作思路


研报不是把指标翻译成句子，而是围绕一个经营问题建立判断链。每章、每节都按以下顺序展开：

1. 先给本节结论：回答“这个指标说明企业处在什么状态”。
2. 再给数据证据：引用输入数据中的金额、占比、增速、排名、客户数、均价、集中度或图表事实。
3. 再做结构解释：说明该现象来自产品结构、市场结构、客户结构、价格带、竞争格局、订单连续性或准入条件中的哪一类因素。
4. 最后落到经营含义：说明企业应巩固、加码、修复、观察还是试单，并给出下一步动作。

每个章节开头必须先写一段总述，说明本章回答什么问题、为什么分析这些指标、这些指标会影响后续什么决策。不能只写一句“本章分析……”。总述应包含本章的分析对象、分析维度和管理用途。

每个小节的常规结构为“总括判断段 + 证据解释段 + 行动建议段”。如果内容较复杂，可增加一段风险或例外情况说明。不要把多个判断挤在一个长句里，也不要把一组结论写成仅用分号连接的流水句。

总结章必须先写2段综合判断，再分为“核心结论、风险提示、战略建议”。每个分点都应是一个独立小段：加粗短标题 + 2-4句解释。禁止输出只有一句话的编号清单。

## 核心原则

1. 只输出分析段落，不输出 Word XML、不输出表格、不输出图表。
2. 不发明数据，所有数字均来自输入的结构化数据。
3. 不写空话，每个判断句后面必须紧跟数据、指标或表格事实。
4. 不堆砌形容词，用数据本身说话。
5. 每个小节输出 2-4 段，每段 3-5 句话；不允许只写一句话。
6. 每个小节必须包含“数据事实—结构判断—经营含义/行动建议”三层信息。
7. 输出要像正式研报正文，不要像接口摘要、字段解释或演示说明。
8. 凡是出现分点，必须采用“短标题 + 完整段落”的方式，不输出压缩编号列表。
9. **锚定市场解读：** 若计算结果显示企业在锚定市场（Anchor Market）份额接近 100% 或无近距离对手，应分析这是否源于 HS 编码定义过细、数据样本限制或该细分领域确实存在极高壁垒，而非简单描述为 “绝对领先”，以保证研报的专业客观性。

## 输入

一份 JSON 格式的结构化数据，包含：

- `company`, `overall_stats`：企业名称和整体统计。
- `product_table`：产品分析表。
- `product_concentration`, `product_concentration_type`：产品集中度。
- `product_trend`：产品年度趋势。
- `country_table`, `country_coverage_type`：国家分布。
- `annual_summary`, `monthly_trend`：年度/月度趋势。
- `market_performance`, `market_growth_classification`：市场表现。
- `market_health`：市场健康度。
- `competition_market`, `competition_hs`, `market_supplier_count`, `cr5`, `cr10`, `market_type`：竞争格局。
- `target_rank`, `target_share`, `target_avg_price`, `price_position`：企业定位。
- `rival_table`, `price_quartiles`：竞争对手和价格。
- `ranked_opportunities`：空白市场机会。
- `top_buyers`, `entry_assessment`：大买家和切入评估。
- `core_findings`, `risks`, `recommendations`：总结。
- `chart_monthly_trend`, `chart_country_amount`, `chart_product_amount` 或其他 PNG/JPEG 图表文件路径。

### metrics 字段规范

生成 `company_metrics.json` 时优先使用以下标准字段，避免 Word 表格留空：

- `overall_stats.total_count` 表示目标企业总交易记录数；`product_table[].txn_count` 只表示该产品/HS 分组内的交易笔数。正文写“总交易记录”时只能引用 `overall_stats.total_count`，不能用某个产品行的 `txn_count` 替代。
- `product_table` 每行使用 `hs`、`desc`、`amount`、`share`、`txn_count`。如果 Excel 已提供 `产品描述中文`、`中文产品描述`、`商品描述中文` 或 `desc_zh`，应优先作为 `desc` 使用；如需保留原文，可额外写 `desc_original`。如果源数据只有 `quantity`，也要同步写入 `txn_count`。
- `country_table` 每行使用 `country`、`amount`、`share`、`customer_count`。`customer_count` 应按目的国统计采购商去重；没有可靠来源时写 `"-"`。
- `market_performance` 每行使用 `country`、`amount`、`share`、`avg_price`、`customer_count`、`growth`。
- `top_buyers` 每行使用 `buyer`、`amount`、`avg_price`、`supplier_count`、`price_gap_vs_target`。`avg_price` 可按 `amount / quantity` 计算；无法计算的字段写 `"-"`，不要省略字段。
- 百分比字段 `share`、`growth`、`top1_customer_share`、`reorder_rate` 推荐使用 0-1 小数；金额和数量使用数值类型。

## 输出格式

按以下格式输出，每个章节用 `###` 标题分隔。只输出段落文字，不要任何其他内容。加粗短标题使用 Markdown 形式 `**短标题。**`，不要直接输出 Word XML。

章节标题必须与 TeX 模板保持一致：全文为五个一级章节；第一章下的 `1.1 核心出口产品` 是父级结构标题，正文写在 `1.1.1`、`1.1.2`、`1.1.3` 三个小节中。不要把第一章拆成额外章节，也不要输出第六章或第七章。

```markdown
### 1 企业贸易全景概览
（2段章首导语。说明本章回答企业“卖什么、卖到哪、卖了多少”的经营问题，并点明产品结构、市场覆盖、月度体量会影响资源配置、市场优先级和风险识别。）

### 1.1.1 核心出口产品识别
（先写本节总括判断，再写2-4段。内容包括：涉及HS编码总数、产品覆盖类别、核心产品识别标准。引用Top3产品的金额和占比，并说明其对收入结构、资源配置、后续市场判断的含义。）

### 1.1.2 产品销售贡献分析
（2-3段。判断产品集中度类型，引用Top1/Top3占比数据。说明集中度对业务韧性、供应链、价格谈判和第二增长曲线的含义。不要只复述表格排名。）

### 1.1.3 产品结构变化趋势
（2-3段。描述主要HS编码的年度金额变化。如有新增或退出产品，简要提及。进一步判断变化来自主力品类放量、品类替代、价格变化还是客户结构变化，并说明后续应监测哪些品类。）

### 1.2 覆盖市场分布
（2-3段。描述市场覆盖数量，引用Top3市场占比。说明市场覆盖类型，并分析头部市场、中腰部市场、尾部市场分别承担的经营角色。）

### 1.3 整体出口体量
（2-3段。描述年度出口总金额、交易笔数、同比增速。提及活跃月份和持续性，并判断增长是连续经营能力、集中大单还是阶段性波动。）

### 2 既有市场表现评估
（2段章首导语。说明本章回答既有市场“哪些应巩固、哪些应加码、哪些需要修复”的问题，并点明金额、均价、增速、客户集中度、复购率的管理用途。）

### 2.1 各市场出货表现
（2-3段。概述各市场出货金额差异，引用头部市场数据和均价范围。按规模型、利润型、试探型市场给出解释。）

### 2.2 增长趋势分析
（3-4段。按增速分类市场，引用增长最快和下滑的市场及其增速。解释增速背后的可能来源，如客户扩张、单价变化、产品结构变化或一次性订单。对高增长市场说明是否适合加码，对下滑市场说明需要复盘什么。）

### 2.3 市场健康度
（3-4段。从客户集中度、月均交易、价格波动、复购率四个维度评估主要市场。引用各市场具体指标，并给出客户维护、风险预警和复购提升动作。）

### 3 竞争格局与自身定位
（2段章首导语。说明本章回答企业在核心市场中处于什么竞争位置，并点明供应商集中度、排名、份额、均价和客户数如何影响份额提升路径。）

### 3.1 市场竞争总览
（2-3段。说明目标市场和HS编码、供应商总数、CR5/CR10、格局类型。引用企业自身份额和排名，并判断该竞争格局下份额提升应依靠价格、客户覆盖还是交付稳定性。）

### 3.2 竞争对手排行
（2-3段。介绍Top3竞争对手的规模、份额、客户数。对比目标企业与头部的份额差距、均价差距、客户数差距，并解释各差距对应的经营含义。）

### 3.3 价格竞争分析
（2-3段。说明市场均价、四分位分布，对比企业均价。判断定价策略定位，并按低价切入、中位竞争、高价差异化三种路径提出适用条件。）

### 3.4 自身定位总结
（3段。综合份额、价格、客户数，总结竞争定位。第一段给定位，第二段解释优势和短板，第三段提出短期客户动作、中期产品动作和长期市场动作。）

### 4 空白市场机会与买家挖掘
```

## 文风红线

避免把以下词当成空洞口号使用：显著、赋能、驱动、深度、全面、重要、关键、有效、大力、积极、强化、深化、助力、支撑、引领、打造、聚焦、着力、扎实、深入推进、持续优化、高质量发展。若必须使用，后面必须紧跟具体对象、数据和动作。

正确示范：

- `Top3产品合计占比87.5%，产品结构集中`：判断加数据。
- `Top1产品占比63.2%，为最大单一品类`：具体。
- `同比增长26.0%，为增速最高的核心市场`：引用数字。
- `产品结构高度集中`：没有数字支撑的判断，不好。
- `增长显著`：空洞形容词，不好。

每条判断后必须紧跟数据证据。不使用感叹号。不使用“我们认为”“可以看出”等主观措辞。
